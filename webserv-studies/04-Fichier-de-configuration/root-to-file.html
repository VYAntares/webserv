04 — Configuration — Résolution de chemin

Du root au fichier final

Une fois la location choisie, le serveur construit le chemin disque par substitution : root + (URI − préfixe). Puis stat() décide si c'est un fichier, un répertoire, ou rien du tout.

Le pipeline complet en 3 étapes

Quand une requête arrive, le routing a déjà sélectionné la location gagnante. Ce qui suit est la traduction de l'URI en chemin réel sur le disque, puis la décision de ce qu'on fait de ce chemin.

🔑Ce n'est pas une addition — c'est une substitution. Le préfixe de la location disparaît et est remplacé par le root.
📞stat() est obligatoire sur chaque chemin construit. On ne suppose jamais ce que contient le disque.
02
root + (URI − préfixe)
La substitution en deux exemples concrets

path_builder.cpp
c++98 
std::string buildPath(const LocationConfig& loc, const std::string& uri)
{
    // ① Retirer le préfixe de la location gagnante
    std::string residual = uri.substr(loc.path.size());
    //   uri = "/kapouet/pouic/toto/pouet", loc.path = "/kapouet"
    //   → residual = "/pouic/toto/pouet"

    // ② Coller root + résidu en évitant le double slash
    const std::string& r = loc.root;
    if (!r.empty() && r.back() == '/' &&
        !residual.empty() && residual[0] == '/')
        return r + residual.substr(1);
    if (!r.empty() && r.back() != '/' &&
        !residual.empty() && residual[0] != '/')
        return r + "/" + residual;
    return r + residual;
}
Cas spécial — URI == préfixe exact

Si l'URI est exactement égale au préfixe de la location, le résidu est une chaîne vide "".

uri = /kapouet, loc.path = /kapouet
→ residual = ""
→ chemin = /tmp/www

/tmp/www est un répertoire — la logique répertoire (index / autoindex / 403) s'applique.

Attention — la formule vient du sujet

Le sujet cite explicitement : « if URL /kapouet is rooted to /tmp/www, URL /kapouet/pouic/toto/pouet will search for /tmp/www/pouic/toto/pouet ».

La logique est confirmée : le préfixe /kapouet est retiré, la partie restante /pouic/toto/pouet est collée à /tmp/www.

Toute autre interprétation (ajout sans strip, double préfixe…) est fausse.

✂️Le préfixe de la location est soustrait de l'URI — il ne figure jamais dans le chemin final.
🔗Le root de la location gagnante est le seul root qui compte — pas celui du bloc server.
🚫Double slash : si root finit par / et que le résidu commence par /, retirer l'un des deux.
03
stat(chemin_construit)
Appel obligatoire — 3 branches, 5 réponses possibles

Une fois le chemin construit, on ne devine pas ce qu'il y a sur le disque. On appelle stat() et on lit le résultat. Trois cas exhaustifs.

static_handler.cpp
c++98 
int handleStaticRequest(const std::string& path,
                           const LocationConfig& loc)
{
    struct stat st;

    // ① Appel obligatoire — jamais supposer
    if (stat(path.c_str(), &st) == -1)
        return 404;  // chemin introuvable

    // ② Fichier ordinaire → servir directement
    if (S_ISREG(st.st_mode))
        return serveFile(path);

    // ③ Répertoire → 3 sous-cas dans l'ordre
    if (S_ISDIR(st.st_mode))
        return handleDirectory(path, loc);

    // ④ Autre (symlink cassé, device...) → 404 défensif
    return 404;
}

int handleDirectory(const std::string& dir, const LocationConfig& loc)
{
    // Sous-cas 1 : index configuré ?
    if (!loc.index.empty()) {
        std::string idx = joinPath(dir, loc.index);
        struct stat si;
        if (stat(idx.c_str(), &si) == 0 && S_ISREG(si.st_mode))
            return serveFile(idx);
        return 404;  // index configuré mais absent → 404 (pas 403)
    }
    // Sous-cas 2 : autoindex activé ?
    if (loc.autoindex)
        return serveDirectoryListing(dir);
    // Sous-cas 3 : rien de tout ça → 403
    return 403;
}
📋L'ordre des sous-cas répertoire est figé : index → autoindex → 403. Jamais dans l'autre sens.
⚠️Index configuré mais fichier absent = 404, pas 403. Le serveur a cherché un fichier précis qui n'existe pas.
🖥️stat() sur disque est une opération synchrone autorisée sans epoll_wait() — exception explicite du sujet.
04
pièges C++98
Les 3 bugs classiques de cette étape
01
Double slash à la jonction
Si root = "/tmp/www" et le résidu vaut "/pouic/...", la concaténation naïve donne "/tmp/www//pouic/...". Sur Linux stat() l'accepte, mais c'est incorrect et peut poser problème ailleurs. Toujours normaliser la jonction.
02
URI == préfixe exact → résidu vide
Si l'URI est exactement égale au préfixe (/kapouet == /kapouet), uri.substr(prefix.size()) retourne "". Le chemin final est juste root. C'est souvent un répertoire — ne pas oublier d'appeler quand même stat() puis handleDirectory().
03
Ne jamais supposer, toujours stat()
Même si l'URI finit par .html ou .jpg, ne pas supposer que c'est un fichier. Il peut ne pas exister, être un répertoire, ou être un lien cassé. stat() est la seule source de vérité — toujours appeler.
joinPath() — utilitaire safe
c++98 
std::string joinPath(const std::string& root, const std::string& residual)
{
    if (root.empty())
        return residual;
    if (residual.empty())
        return root;

    bool rootSlash     = (root.back()      == '/');
    bool residualSlash = (residual.front() == '/');

    if (rootSlash && residualSlash)       // "root/" + "/res" → "root/res"
        return root + residual.substr(1);
    if (!rootSlash && !residualSlash)    // "root" + "res"  → "root/res"
        return root + "/" + residual;
    return root + residual;               // cas normal — pas de conflit
}
⚠️
Path traversal : si le résidu contient ../, le chemin résolu peut sortir du root. Valider que le chemin final commence toujours par le root configuré (ou utiliser realpath() + vérification de préfixe).

# ④ Chemin inexistant → 404

curl -i http://localhost:8080/kapouet
Ce que tu dois observer

Le test ⑥ est le plus important : index configuré mais fichier absent. La réponse doit être 404 — pas 403, pas 500. Si tu obtiens 403, c'est que tu as sauté la vérification stat() sur le fichier index.

Le test ③ vérifie que l'URI se terminant par / ne déclenche pas un stat() sur un fichier imaginaire.

Checklist de validation

✅ Fichier existant → 200
✅ URI == préfixe → répertoire correctement détecté
✅ autoindex → listing HTML généré
✅ chemin absent → 404
✅ répertoire sans index/autoindex → 403
✅ index configuré + absent → 404
✅ pas de double slash dans les chemins loggés

🧪Le test ⑥ (index absent → 404 pas 403) est la discrimination entre une implémentation correcte et une approximation.
🔎Logger le chemin construit à chaque requête (std::cerr << "path: " << path) permet de vérifier visuellement que la substitution est juste.
🗣️À la soutenance : l'évaluateur peut demander ce que retourne le serveur pour /kapouet exactement — connaître la réponse à l'oral prouve la maîtrise.
