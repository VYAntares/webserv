04 — Sujet IV.3

Ce que la config doit absolument contenir

Version courte et directe: uniquement les extraits minimum de ton fichier, avec l’explication associée.

01
interface:port
Définir tous les couples d’écoute
webserv.conf
minimum 
server {
    listen 8080;
    server_name localhost;
    ...
}

server {
    listen 8081;
    ...
}

Ton programme doit pouvoir écouter plusieurs ports/sites. Ici: un serveur sur 8080, un autre sur 8081.

02
error pages
Pages d’erreur par défaut
webserv.conf
minimum 
error_page 404 /errors/404.html;
error_page 500 /errors/500.html;

Quand une erreur se produit, le serveur envoie ta page personnalisée au lieu d’une page générique.

03
client_max_body_size
Taille maximale du body client
webserv.conf
minimum 
server {
    ...
    client_max_body_size 10m;
}

server {
    ...
    client_max_body_size 1m;
}

Permet de limiter les uploads/requêtes lourdes par serveur. Ici 10 Mo pour le site principal, 1 Mo pour le second.

04
règles par route
Méthodes, redirection, root, listing, index, upload
webserv.conf
méthodes + root + index + autoindex 
location / {
    root            /var/www/html;
    index           index.html;
    allowed_methods GET;
    autoindex       off;
}

location /api {
    root            /var/www/api;
    allowed_methods GET POST DELETE;
    autoindex       off;
}

accepted methods: contrôlées par allowed_methods.

directory mapping: root fixe la base disque pour la route.

default file: index sert le fichier d’entrée d’un dossier.

directory listing: autoindex on/off active ou non le listing.

webserv.conf
redirection 
location /old-page {
    return 301 /new-page;
}

HTTP redirection: ici /old-page redirige en 301 vers /new-page.

webserv.conf
upload autorisé 
location /upload {
    root            /var/www;
    allowed_methods POST;
    upload_store    /tmp/uploads;
}

upload files: route dédiée en POST + répertoire de stockage explicite.

Tu couvres exactement les points demandés par le sujet IV.3, sans ajouter de règles inutiles.
