Concept — I/O Multiplexing

La boucle événementielle
Intégrer epoll dans le serveur

On part du code main.cpp actuel — un serveur busy-wait qui brûle du CPU — et on le transforme fonction par fonction en un serveur event-driven avec epoll. Chaque étape montre l'avant, explique le pourquoi, puis montre l'après.

01
Le code actuel
Ce qui ne va pas — et pourquoi
Le problème

Le serveur actuel est non-bloquant (on a mis O_NONBLOCK), mais il n'utilise pas epoll. Il tourne dans une boucle while(true) qui appelle accept() en continu.

Quand personne ne se connecte, accept() retourne EAGAIN et on re-boucle immédiatement. Le CPU tourne à 100% pour rien. C'est du busy-waiting.

De plus, handle_client() fait un do/while avec recv() qui boucle sur EAGAIN — encore du busy-wait. Et on ne gère qu'un seul client à la fois.

Ce qu'on veut
  • 0% CPU au repos — epoll_wait() dort quand rien ne se passe
  • Plusieurs clients simultanés — pas de traitement séquentiel
  • Jamais de recv/send sans epoll — respecter la règle du sujet
  • Transition EPOLLIN → EPOLLOUT — lire la requête, puis envoyer la réponse
  • Un seul point de blocage — epoll_wait() dans le main
main.cpp
Avant — busy-wait 
// La boucle actuelle — le problème est ici

while (true) {
    int client_fd = accept_next_client(server_fd);  // ← EAGAIN en boucle = CPU 100%
    if (client_fd == -1 || client_fd == -2)
        continue;                                 // ← on re-boucle immédiatement
    handle_client(client_fd);                     // ← un seul client à la fois
}

// Et dans handle_client() :
do {
    n = recv(client_fd, buf, sizeof(buf) - 1, 0);
} while (n == -1 && errno == EAGAIN);   // ← busy-wait sur recv aussi !

« It must be non-blocking and use only 1 poll() (or equivalent) for all the I/O operations between the clients and the server (listen included). »

— webserv.pdf, IV.1 Requirements
02
main()
Créer epoll et construire la boucle
Ce qu'on va modifier

Le main() doit faire 3 choses nouvelles :

  • Créer l'instance epoll — epoll_create1(0), une seule fois au démarrage
  • Enregistrer le listen_fd — epoll_ctl(ADD) avec EPOLLIN pour détecter les connexions entrantes
  • Remplacer la boucle — epoll_wait() au lieu du while(true) + accept() direct
main.cpp — main()
Avant 
int main() {
    int server_fd = create_server_socket();
    if (server_fd == -1) return 1;

    if (bind_address(server_fd, PORT) == -1) return 1;

    if (listen(server_fd, BACKLOG) == -1)
        { perror("listen"); return 1; }

    // Boucle busy-wait :
    while (true) {
        int client_fd = accept_next_client(server_fd);
        if (client_fd == -1 || client_fd == -2) continue;

    }

main.cpp — main()
Après — avec epoll 
#include <sys/epoll.h>
#include <csignal>

static const int MAX_EVENTS = 64;

int main() {
    signal(SIGPIPE, SIG_IGN);  // ignorer SIGPIPE — send() sur socket fermé ne crash plus

    int server_fd = create_server_socket();
    if (server_fd == -1) return 1;
    if (bind_address(server_fd, PORT) == -1) return 1;
    if (listen(server_fd, BACKLOG) == -1)
        { perror("listen"); return 1; }

    // ── NOUVEAU : créer l'instance epoll ──────────────────
    int epfd = epoll_create1(0);
    if (epfd == -1) { perror("epoll_create1"); return 1; }

    // ── NOUVEAU : enregistrer le listen_fd dans epoll ────
    // On veut être notifié quand un client se connecte (EPOLLIN)
    struct epoll_event ev;
    ev.events  = EPOLLIN;
    ev.data.fd = server_fd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, server_fd, &ev);

    // ── NOUVEAU : la boucle événementielle ───────────────
    struct epoll_event events[MAX_EVENTS];

    while (true) {
        // LE seul point de blocage — dort quand rien ne se passe
        int n = epoll_wait(epfd, events, MAX_EVENTS, -1);

        for (int i = 0; i < n; i++) {
            int fd = events[i].data.fd;

            if (fd == server_fd) {
                // Connexion entrante → accept
                accept_next_client(server_fd, epfd);
            }
            else if (events[i].events & EPOLLIN) {
                // Client prêt en lecture → recv
                handle_read(fd, epfd);
            }
            else if (events[i].events & EPOLLOUT) {
                // Client prêt en écriture → send
                handle_write(fd, epfd);
            }

            if (events[i].events & (EPOLLERR | EPOLLHUP)) {
                // Erreur ou déconnexion → nettoyer
                epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
                close(fd);
            }
        }
    }
}
💤 epoll_wait(-1) dort indéfiniment s'il n'y a rien. 0% CPU au repos. Le kernel te réveille uniquement quand un fd est prêt.
🔀 Un seul point de blocage. Avant, le blocage était dispersé (accept, recv). Maintenant tout passe par epoll_wait().
👥 Plusieurs clients. epoll_wait() peut retourner plusieurs événements. On les traite tous dans la boucle for.
03
accept_next_client()
Enregistrer le nouveau client dans epoll
Ce qu'on va modifier

L'ancienne fonction acceptait un client et le retournait. La nouvelle doit en plus :

  • Enregistrer le client_fd dans epoll — epoll_ctl(ADD, client_fd, EPOLLIN)
  • Ne plus retourner le fd au main — c'est epoll qui le gérera désormais
  • Supprimer le test EAGAIN — epoll garantit que accept() va réussir (il nous a dit EPOLLIN)
main.cpp — accept_next_client()
Avant 
int accept_next_client(int server_fd) {
    struct sockaddr_in client_addr;
    socklen_t len = sizeof(client_addr);
    int client_fd = accept(server_fd,
                           (struct sockaddr*)&client_addr, &len);
    if (client_fd == -1) {
        if (errno == EAGAIN) return -2;  // ← plus besoin de ça
        perror("accept"); return -1;
    }

    fcntl(client_fd, F_SETFL, fcntl(client_fd, F_GETFL) | O_NONBLOCK);
    return client_fd;  // ← retournait le fd au main
}
main.cpp — accept_next_client()
Après — avec epoll 
// Nouvelle signature : on reçoit epfd pour enregistrer le client
void accept_next_client(int server_fd, int epfd) {
    struct sockaddr_in client_addr;
    socklen_t len = sizeof(client_addr);
    int client_fd = accept(server_fd,
                           (struct sockaddr*)&client_addr, &len);

    if (client_fd == -1) { perror("accept"); return; }

    // Mettre en non-bloquant (identique à avant)
    fcntl(client_fd, F_SETFL, fcntl(client_fd, F_GETFL) | O_NONBLOCK);

    // ── NOUVEAU : enregistrer le client dans epoll ──────
    // On veut savoir quand le client envoie des données (EPOLLIN)
    // PAS de recv() ici ! On attend le prochain epoll_wait()
    struct epoll_event ev;
    ev.events  = EPOLLIN;       // surveiller en lecture
    ev.data.fd = client_fd;    // stocker le fd pour le retrouver
    epoll_ctl(epfd, EPOLL_CTL_ADD, client_fd, &ev);

    std::cout << "[+] client fd=" << client_fd
              << " from " << inet_ntoa(client_addr.sin_addr) << "\n";
}
💡
Pourquoi pas de recv() ici ? Le client vient de se connecter (TCP handshake OK), mais il n'a peut-être pas encore envoyé sa requête HTTP. On l'enregistre dans epoll et on retourne dans la boucle. Quand le client enverra ses données, epoll_wait() nous réveillera avec EPOLLIN sur ce fd.
04
handle_recv() / handle_send()
Séparer lecture et écriture — en mode Edge Triggered
Le changement fondamental

L'ancienne handle_client() faisait tout d'un coup : recv + parse + send + close. C'est impossible avec epoll car on ne peut pas savoir si send() va réussir immédiatement.

On doit séparer en deux fonctions :

  • handle_recv(fd, epfd) — appelée quand EPOLLIN arrive. Draine toutes les données disponibles, puis passe le fd en EPOLLOUT | EPOLLET
  • handle_send(fd, epfd) — appelée quand EPOLLOUT arrive. Envoie la réponse entière en boucle, puis ferme le fd
Choix : Level Triggered (LT) ou Edge Triggered (ET) ?

epoll supporte deux modes de déclenchement. Ce choix change radicalement comment on écrit le code.

🔔
LT — Level Triggered (défaut)

epoll te réveille tant que des données sont disponibles dans le buffer. Un seul recv() suffit : si tu n'as pas tout lu, epoll te renotifiera au prochain tour. Plus simple à écrire, mais moins efficace.

⚡
ET — Edge Triggered (EPOLLET)

epoll te réveille une seule fois lors du changement d'état (nouvelles données arrivées). Si tu ne lis pas tout le buffer en une fois, tu ne seras plus notifié. Oblige à boucler jusqu'à EAGAIN — mais une seule interruption noyau par événement.

Choix retenu : ET pour EPOLLOUT, drain complet pour EPOLLIN. Pour handle_recv, on boucle jusqu'à EAGAIN pour vider tout le buffer d'un coup. Pour handle_send, on enregistre EPOLLOUT | EPOLLET et on boucle jusqu'à envoyer tout — si EAGAIN survient, on se ré-enregistre et on reprend au prochain tour.

main.cpp — handle_client()
Avant — tout d'un bloc 
void handle_client(int client_fd) {
    char buf[4096];
    int n;

    do {
        n = recv(client_fd, buf, sizeof(buf) - 1, 0);
    } while (n == -1 && errno == EAGAIN);  // busy-wait !

    if (n <= 0) { close(client_fd); return; }
    buf[n] = '\0';

    const char* response = "HTTP/1.1 200 OK\r\n...";
    send(client_fd, response, strlen(response), 0);  // sans vérifier EPOLLOUT !
    close(client_fd);
}
main.cpp — handle_recv()
Après — lecture ET (drain jusqu'à EAGAIN) 
void handle_recv(int fd, int epfd) {
    std::string request;

    // Boucle ET : on vide TOUT le buffer d'un coup.
    // En LT un seul recv() suffirait — ici on doit aller jusqu'à EAGAIN.
    while (true) {
        char buf[4096];
        ssize_t n = recv(fd, buf, sizeof(buf) - 1, 0);

        if (n < 0) {
            if (errno == EAGAIN || errno == EWOULDBLOCK)
                break;  // buffer kernel vide → on a tout lu, on sort
            epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
            close(fd); return;
        }
        if (n == 0) {  // client déconnecté
            epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
            close(fd); return;
        }
        buf[n] = '\0';
        request += buf;
    }
    std::cout << request;  // afficher la requête (debug)

    // Requête lue — passer en EPOLLOUT | EPOLLET pour l'envoi
    struct epoll_event ev;
    ev.events  = EPOLLOUT | EPOLLET;  // ET : une seule notif quand le buffer se libère
    ev.data.fd = fd;
    epoll_ctl(epfd, EPOLL_CTL_MOD, fd, &ev);  // MOD : fd déjà enregistré
}
main.cpp — handle_send()
Après — écriture ET (boucle jusqu'à complet) 
static const char* response =
    "HTTP/1.1 200 OK\r\n"
    "Content-Type: text/html\r\n"
    "Content-Length: 22\r\n"
    "Connection: close\r\n"
    "\r\n"
    "<h1>Hello World!</h1>\n";

void handle_send(int fd, int epfd) {
    size_t total = strlen(response);
    size_t sent  = 0;

    // Boucle ET : envoyer TOUT d'un coup.
    // Si send() retourne EAGAIN, le buffer noyau est plein →
    // on reste en EPOLLOUT | EPOLLET et on reprendra au prochain tour.
    while (sent < total) {
        ssize_t n = send(fd, response + sent, total - sent, 0);
        if (n < 0) {
            if (errno == EAGAIN || errno == EWOULDBLOCK) {
                // buffer plein : se ré-enregistrer, attendre le prochain EPOLLOUT
                struct epoll_event ev;
                ev.events  = EPOLLOUT | EPOLLET;
                ev.data.fd = fd;
                epoll_ctl(epfd, EPOLL_CTL_MOD, fd, &ev);
                return;  // reprendre au prochain EPOLLOUT
            }
            break;  // erreur réelle
        }
        sent += n;
    }
    // Tout envoyé — nettoyer
    epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
    close(fd);
}
⚠️
EPOLL_CTL_MOD, pas ADD ! Le client_fd est déjà dans epoll (ajouté lors de l'accept). On utilise EPOLL_CTL_MOD pour changer les événements surveillés (de EPOLLIN à EPOLLOUT | EPOLLET). EPOLL_CTL_ADD sur un fd déjà enregistré retournerait EEXIST.
05
Et le CGI ?
Comment ça s'intègre — aperçu
On verra les détails plus tard — voici le principe

Quand une requête déclenche un CGI (fork() + execve() d'un script Python/PHP), le processus fils écrit sa sortie dans un pipe. Ce pipe est un fd comme un autre.

Le principe est identique aux sockets clients :

  • Enregistrer le pipe dans epoll — epoll_ctl(ADD, pipe_fd, EPOLLIN)
  • Attendre EPOLLIN — quand le CGI produit de la sortie, epoll nous réveille
  • Lire le pipe — read(pipe_fd) uniquement après EPOLLIN
  • Transmettre au client — une fois la réponse CGI complète, passer le client_fd en EPOLLOUT

Le pipe du CGI est aussi en mode non-bloquant. Jamais de read() sur un pipe sans que epoll_wait() l'ait confirmé. Même règle que pour les sockets.

📌
Le CGI aura son propre chapitre. Pour l'instant, retiens juste : un pipe CGI se gère exactement comme un socket client dans la boucle événementielle — enregistrer dans epoll, lire/écrire après notification, nettoyer à la fin. La seule différence est fork() + waitpid(WNOHANG) pour le processus fils.
06
Code final
Le serveur complet avec epoll
Ce qui a changé en résumé
  • create_server_socket() — inchangée (socket + SO_REUSEADDR + O_NONBLOCK)
  • bind_address() — inchangée
  • accept_next_client() — reçoit epfd, enregistre le client dans epoll avec EPOLLIN
  • handle_client() — supprimée, remplacée par handle_read() et handle_write()
  • main() — crée epoll, enregistre listen_fd, boucle sur epoll_wait()
main.cpp
Code final complet 
#include <sys/socket.h>
#include <sys/epoll.h>
#include <arpa/inet.h>
#include <unistd.h>
#include <fcntl.h>
#include <csignal>
#include <cstring>
#include <cerrno>
#include <iostream>

static const int PORT       = 8080;
static const int BACKLOG    = 10;
static const int MAX_EVENTS = 64;

static const char* RESPONSE =
    "HTTP/1.1 200 OK\r\n"
    "Content-Type: text/html\r\n"
    "Content-Length: 22\r\n"
    "Connection: close\r\n"
    "\r\n"
    "<h1>Hello World!</h1>\n";

// ── Crée le socket serveur (inchangé) ─────────────────
int create_server_socket() {
    int fd = socket(AF_INET, SOCK_STREAM, 0);
    if (fd == -1) { perror("socket"); return -1; }

    int opt = 1;
    setsockopt(fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
    fcntl(fd, F_SETFL, fcntl(fd, F_GETFL) | O_NONBLOCK);
    return fd;
}

// ── Bind IP:port (inchangé) ───────────────────────────
int bind_address(int fd, int port) {
    struct sockaddr_in addr;
    memset(&addr, 0, sizeof(addr));
    addr.sin_family      = AF_INET;
    addr.sin_port        = htons(port);
    addr.sin_addr.s_addr = INADDR_ANY;

    if (bind(fd, (struct sockaddr*)&addr, sizeof(addr)) == -1)
        { perror("bind"); return -1; }
    return 0;
}

// ── Accept : enregistre le client dans epoll ─────────
void accept_next_client(int server_fd, int epfd) {
    struct sockaddr_in client_addr;
    socklen_t len = sizeof(client_addr);
    int client_fd = accept(server_fd,
                           (struct sockaddr*)&client_addr, &len);
    if (client_fd == -1) { perror("accept"); return; }

    fcntl(client_fd, F_SETFL, fcntl(client_fd, F_GETFL) | O_NONBLOCK);

    struct epoll_event ev;
    ev.events  = EPOLLIN;
    ev.data.fd = client_fd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, client_fd, &ev);

    std::cout << "[+] client fd=" << client_fd
              << " from " << inet_ntoa(client_addr.sin_addr) << "\n";
}

// ── Lecture : EPOLLIN → recv, puis passe en EPOLLOUT ──
void handle_read(int fd, int epfd) {
    char buf[4096];
    ssize_t n = recv(fd, buf, sizeof(buf) - 1, 0);

    if (n <= 0) {
        epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
        close(fd);
        return;
    }

    buf[n] = '\0';
    std::cout << buf;

    // Passer de lecture → écriture
    struct epoll_event ev;
    ev.events  = EPOLLOUT;
    ev.data.fd = fd;
    epoll_ctl(epfd, EPOLL_CTL_MOD, fd, &ev);
}

// ── Écriture : EPOLLOUT → send, puis close ───────────
void handle_write(int fd, int epfd) {
    send(fd, RESPONSE, strlen(RESPONSE), 0);

    epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
    close(fd);
}

// ── La boucle événementielle ─────────────────────────
int main() {
    signal(SIGPIPE, SIG_IGN);

    int server_fd = create_server_socket();
    if (server_fd == -1) return 1;
    if (bind_address(server_fd, PORT) == -1) return 1;
    if (listen(server_fd, BACKLOG) == -1)
        { perror("listen"); return 1; }

    // Créer epoll + enregistrer le listen socket
    int epfd = epoll_create1(0);
    if (epfd == -1) { perror("epoll_create1"); return 1; }

    struct epoll_event ev;
    ev.events  = EPOLLIN;
    ev.data.fd = server_fd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, server_fd, &ev);

    std::cout << "Serveur sur http://localhost:" << PORT << "\n";

    // Boucle événementielle
    struct epoll_event events[MAX_EVENTS];

    while (true) {
        int n = epoll_wait(epfd, events, MAX_EVENTS, -1);

        for (int i = 0; i < n; i++) {
            int fd = events[i].data.fd;

            if (events[i].events & (EPOLLERR | EPOLLHUP)) {
                epoll_ctl(epfd, EPOLL_CTL_DEL, fd, NULL);
                close(fd);
                continue;
            }

            if (fd == server_fd)
                accept_next_client(server_fd, epfd);
            else if (events[i].events & EPOLLIN)
                handle_read(fd, epfd);
            else if (events[i].events & EPOLLOUT)
                handle_write(fd, epfd);
        }
    }

    close(epfd);
    close(server_fd);
    return 0;
}
💤 0% CPU au repos. epoll_wait(-1) dort. Plus de busy-wait.
👥 Multi-clients. Chaque client est un fd dans epoll. Ils sont tous gérés en parallèle dans la même boucle.
✅ Règle respectée. Aucun recv/send sans epoll_wait() avant. Aucun errno vérifié après.
🔀 Transition claire. EPOLLIN → recv → EPOLL_CTL_MOD(EPOLLOUT) → send → close. Le cycle de vie d'une requête.
