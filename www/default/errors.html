Codes d'erreur — « Your HTTP response status codes must be accurate »

Chaque test provoque volontairement une erreur et vérifie le code exact de la réponse. Les erreurs 400 (requête malformée) et la robustesse bas niveau sont couvertes par tests/run_all_tests.sh (impossibles à générer depuis un navigateur).

Pages d'erreur

À vérifier visuellement : /nexiste-pas doit afficher la page 404 personnalisée (demo.conf, via error_page) ; avec default.conf c'est la page par défaut générée par le serveur. /cgi-bin/crash.py → page 50x personnalisée.

Toutes les pages d'erreur

🔴 400 Bad Request

Requête HTTP malformée (header sans :).

📄 Page 400

curl :

printf 'GET / HTTP/1.1\r\nHost: x\r\nBadHeader\r\n\r\n' | nc 100.124.128.83 8080

🔒 403 Forbidden

Accès interdit à un dossier sans index.

📄 Page 403

curl :

curl -i http://100.124.128.83:8080/private/

❓ 404 Not Found

Ressource inexistante sur le serveur.

📄 Page 404

curl :

curl -i http://100.124.128.83:8080/nexiste-pas

🚫 405 Method Not Allowed

Méthode interdite sur cette route.

📄 Page 405

curl :

curl -i -X POST http://100.124.128.83:8080/

📦 413 Payload Too Large

Body dépasse client_max_body_size de la route.

📄 Page 413

curl :

curl -i -X POST -d "$(python3 -c "print('x'*200)")" http://100.124.128.83:8080/tiny/limit.txt

📏 431 Header Too Large

Un header dépasse 32 Ko.

📄 Page 431

curl :

curl -i -H "X-Big: $(python3 -c "print('A'*40000)")" http://100.124.128.83:8080/

💥 500 Internal Server Error

Le CGI crash (exit code ≠ 0).

📄 Page 500

curl :

curl -i http://100.124.128.83:8080/cgi-bin/crash.py

🔧 501 Not Implemented

Méthode HTTP inconnue du serveur.

📄 Page 501

curl :

curl -i -X PATCH http://100.124.128.83:8080/

🌉 502 Bad Gateway

Le CGI est tué par un signal (SIGSEGV).

📄 Page 502

curl :

curl -i http://100.124.128.83:8080/cgi-bin/sigsegv.py

⏱️ 504 Gateway Timeout

Le CGI met trop longtemps (>30s).

📄 Page 504

curl :

curl -i --max-time 35 http://100.124.128.83:8080/cgi-bin/slow.py
