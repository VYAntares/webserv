Stress — « Your server must remain operational at all times »

Envoie des rafales de requêtes concurrentes (mélange statique / 404 / autoindex / CGI) et vérifie que tout est répondu et que le serveur reste disponible ensuite. 

—

Tests avec siege (terminal)

siege est l'outil de benchmarking utilisé en soutenance. Installer : sudo apt install siege. Clic sur une commande = copie automatique.

📋 Test officiel soutenance — benchmark

-b = mode benchmark (pas de pause). Le barème exige > 99.5% de disponibilité.

siege -b -c50 -t30s http://127.0.0.1:8080/default/index.html

✅ Attendu : Availability > 99.5%, Failed: 0

📋 Variante — 100 clients, 30s

Pousser plus fort avec 100 connexions simultanées.

siege -b -c100 -t30s http://127.0.0.1:8080/default/index.html

✅ Attendu : Availability > 99.5%

📋 Avec délai — -d (recommandé grille)

La grille conseille -d (délai entre reconnexions) pour un test plus réaliste.

siege -c50 -d1 -r50 http://127.0.0.1:8080/default/index.html

✅ Attendu : Availability: 100%, Failed: 0

📋 Test longue durée — infini

La grille dit : « utiliser siege indéfiniment sans redémarrer le serveur ». Ctrl+C pour arrêter.

siege -b -c30 http://127.0.0.1:8080/default/index.html

✅ Attendu : le serveur ne crashe jamais. Ctrl+C après plusieurs minutes, Availability: 100%

📋 Mix URLs + benchmark

Fichier d'URLs variées, mode benchmark.

echo -e "http://127.0.0.1:8080/default/index.html\nhttp://127.0.0.1:8080/files/hello.txt\nhttp://127.0.0.1:8080/nexiste-pas\nhttp://127.0.0.1:8080/files/\nhttp://127.0.0.1:8080/cgi-bin/hello.py" > /tmp/urls.txt && siege -b -c30 -t30s -f /tmp/urls.txt

✅ Attendu : Availability > 98% (le 404 est normal)

Rappel grille de soutenance : availability > 99.5% pour un GET simple avec siege -b. Pas de fuite mémoire (surveiller avec htop). Pas de connexions bloquées. Le serveur doit tourner indéfiniment sans crash.
