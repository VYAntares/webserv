CGI — GET, POST, variables d'environnement, erreurs

Le serveur exécute les CGI selon l'extension (cgi_pass .py /usr/bin/python3, .sh /usr/bin/bash en demo.conf), transmet la requête complète (query string, headers en HTTP_*, body sur stdin jusqu'à EOF) et parse la sortie CGI (Status:, Content-Type:).

GET — Python

🎲 Citations aléatoires

Génère une citation de programmeur célèbre. Change le thème couleur via ?cyan, ?purple, ?green, ?orange.

fortune.py thème violet

🎨 Générateur de palettes

Palette de couleurs aléatoire avec valeurs RGB et hex. Ajoute ?n=8 pour 8 couleurs.

palette.py 8 couleurs

GET — Bash

🖥️ Infos système

Informations sur la machine qui fait tourner le serveur : hostname, uptime, date, shell.

sysinfo.sh

🕐 Horloge mondiale

Horloge qui se rafraîchit chaque seconde. Change de fuseau : ?offset=1 (Paris), ?offset=-5 (NY), ?offset=9 (Tokyo).

clock.sh (UTC) Paris

CGI lent → timeout serveur

slow.py dort 35 s, plus que le timeout serveur (30 s). Exigence du sujet : « A request to your server should never hang indefinitely ». Le serveur doit couper la requête (~30 s) sans jamais bloquer les autres clients — ouvrir un autre onglet pendant l'attente : il doit répondre instantanément.
