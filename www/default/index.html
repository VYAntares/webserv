Suite de tests webserv

Pages de démonstration prouvant que le serveur couvre toutes les exigences du sujet. Lancer le serveur avec ./webserv conf/demo.conf puis ouvrir http://localhost:8080/default/.

conf/demo.conf est un sur-ensemble de default.conf : mêmes routes, plus /tiny (test 413), /private (test 403), /redirect302, le CGI shell (.sh), les pages d'erreur personnalisées et un second serveur sur le port 8081. Avec default.conf, ces tests-là échoueront, le reste fonctionne.

Pages de test

GET & HEAD

Site statique, types MIME, index, autoindex, redirections 301/302, HEAD.

POST

Bodies bruts de 1 Ko à 12 Mo, formulaires urlencoded, limite client_max_body_size (413).

Upload (multipart)

Upload de fichiers depuis le navigateur : vos fichiers + fichiers générés de 1 Ko à 12 Mo, multi-fichiers.

DELETE

Lister le dossier d'upload et supprimer les fichiers un par un (204), cycle create→delete→404.

CGI

CGI GET (query string), CGI POST (petits et gros bodies), variables d'environnement, erreurs CGI (500/502/504).

Codes d'erreur

403, 404, 405, 413, 431, 501 + pages d'erreur par défaut et personnalisées.

Stress / disponibilité

Rafales de requêtes concurrentes depuis le navigateur, le serveur doit rester disponible.

Bonus

Cookies & sessions (CGI + fichier de session côté serveur), plusieurs types de CGI (Python + Bash).

Correspondance avec le sujet

  • Fichier de configuration en argument (conf/demo.conf) ou chemin par défaut
  • Serveur non-bloquant, un seul epoll() pour toutes les I/O — voir stress
  • Compatible navigateur : ces pages en sont la preuve
  • Codes de statut HTTP exacts — voir erreurs
  • Pages d'erreur par défaut + personnalisées (error_page)
  • Site entièrement statique servi (cette suite)
  • Upload de fichiers par les clients — voir upload
  • Méthodes GET, POST, DELETE (+ HEAD)
  • Écoute sur plusieurs ports, contenus différents — port 8081
  • Routes configurables : méthodes, redirection, root, autoindex, index, upload_store, CGI par extension
  • Limite de taille de body par route (client_max_body_size)
  • Requêtes chunked dé-chunkées pour le CGI — voir tests/run_all_tests.sh (impossible depuis un navigateur)
  • Bonus : cookies & sessions, plusieurs CGI — bonus + tests/bonus_*.sh

Scripts de test en ligne de commande

bash tests/run_all_tests.sh            # suite complète partie obligatoire (curl)
bash tests/bonus_cookies_sessions.sh   # bonus : cookies + sessions
bash tests/bonus_multi_cgi.sh          # bonus : plusieurs types de CGI
python3 tests/stress_test.py           # stress : 500 requêtes concurrentes
