POST — bodies bruts, tailles croissantes, limites

Chaque test envoie un body brut vers /upload/<nom> : le serveur écrit le fichier dans upload_store (./www/upload) et répond 201 (créé) ou 200 (écrasé). Le fichier est ensuite relu (GET) pour vérifier que le contenu est intact, puis supprimé (DELETE). Les trois derniers tests vérifient client_max_body_size : 10 Mo sur le serveur, 100 octets sur la route /tiny (demo.conf) → 413 au-delà.
